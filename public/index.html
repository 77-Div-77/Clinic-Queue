No training required. No app to install. Just open the browser and start.

Enter the patient name — press Enter. ClinicQ assigns the next token number automatically in under 5 seconds.

Features

Everything a clinic actually needs

Practical features designed around real clinic workflows — not theoretical ones.

⚡

Under 5-Second Registration

Single required field with auto-focus. Enter key submits. Patient added, token assigned, input cleared — all instantly.

🔄

Zero-Refresh Live Updates

Socket.IO pushes every queue change to all connected screens the moment it happens. No polling. No button to click.

📊

Real Wait Time Algorithm

Rolling average of actual consultation durations. After 2 sessions, the estimate is based on real data — not guesswork.

↩

30-Second Undo Window

Called the wrong token? A 30-second undo button lets the receptionist revert the call completely — queue restores perfectly.

🔍

Patient Self-Lookup

Patients type their own token number on the patient screen to see their exact position and live estimated wait time.

📶

Auto-Reconnect on Network Loss

Socket.IO reconnects automatically on any network drop. Full queue state is delivered back within milliseconds of reconnecting.

🔊

Audio Token Announcements

Built-in Web Speech API integration automatically announces token numbers across the clinic's sound system the moment they are called.

Frequently Asked Questions

No app required. The patient view is a regular web page. Patients open the link on their phone browser and the queue updates live — powered by WebSocket technology.
Socket.IO includes automatic reconnection with exponential backoff. When connectivity restores, each client automatically receives the full current queue state within milliseconds. No data is lost.
No. ClinicQ detects MongoDB availability at startup. If it's not reachable, it automatically falls back to in-memory mode — the clinic runs perfectly, data just won't persist across server restarts.
It uses a rolling average of the last 10 actual consultation durations. After 2 completions, real data overrides the manual setting. The algorithm also deducts time already elapsed in the current consultation — so patient #3 sees a shorter wait if the current patient is 8 minutes into a 10-minute average.
Yes. All state lives on the server. Both clients receive identical broadcasts. Since Node.js processes events serially, there are no race conditions — both receptionists see the same queue at all times.
A 30-second undo button appears after every "Call Next". Clicking it reverts the queue completely — the called patient returns to waiting, and all wait times recalculate. After 30 seconds the button disappears permanently.
